Partition

      
Data is split horizontally into N equal slices. Each worker operates on its own slice — no shared memory, no locking.

      
data[0:50]   → Partition 0
data[50:100] → Partition 1
data[100:150]→ Partition 2
data[150:200]→ Partition 3

    

    

      
📋

      
Task

      
One operation × one partition = one Task. The Scheduler assigns tasks to workers; the Executor tracks results and re-assembles partitions.

      
Task(
  op = FILTER,
  partition = P0,
  worker = W1
)

    

    

      
🧠

      
Lazy DAG

      
Calling .filter().map().groupby() records operations, nothing runs. .collect() triggers planning: logical plan → physical DAG → execution.

      
ds.filter(...)   # no-op
  .map(...)      # no-op
  .collect()     # runs!

    

    

      
🔧

      
Scheduler

      
Assigns tasks to workers in round-robin order. Uses multiprocessing.Pool under the hood — tasks run in real parallel subprocesses.

      
worker 0 ← task 0, 4, 8
worker 1 ← task 1, 5, 9
worker 2 ← task 2, 6, 10
worker 3 ← task 3, 7, 11

    

    

      
📦

      
Dill Serialization

      
Lambdas can't be pickled by Python's default pickle. We use dill to serialize closures and lambdas — exactly how PySpark uses cloudpickle.

      
func_blob = dill.dumps(
  lambda r: r["age"] > 30
)
# crosses process boundary safely

    

    

      
🎯

      
Context Manager

      
The Scheduler pool is managed with a context manager — workers start on enter, cleanly terminate on exit, even if an exception is raised.

      
with pipeline_context(4) as ctx:
    result = DataSet(data, ctx)
              .filter(...)
              .collect()
